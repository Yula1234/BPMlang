# BPMlang

BPMlang — экспериментальный статически типизированный язык программирования с шаблонами и простыми абстракциями наподобие C++/Rust, компилируемый в x86-assembler и далее в нативный бинарник (сейчас — 32‑битный Win32)

Репозиторий содержит сам компилятор BPMlang (на C++), стандартную библиотеку и несколько примеров.

## Возможности

- Статическая типизация, ссылки/указатели, `struct`/`impl`.
- Шаблоны (generic‑типы и функции): `vector<T>`, `map<K, V>` и т.п.
- Простейшее ОО‑подобное программирование через `impl` и методы:
  - вызовы вида `obj~method(...)` и `ns::func(...)`.
- Исключения и стек трассировки (`traceback`).
- Стандартная библиотека:
  - `std` — ввод/вывод, строки;
  - `vector`, `forward_list`, `deque`;
  - `std_string`, `string_view`;
  - `hash_map`;
  - `fsys`, `algorithm` и др.

## Пример

```bpm
include "std"
include "vector"

proc main() {
    let a = vector::new<int>();
    a~push(4);
    std::print(a);
}
```

Компилятор превращает этот код в x86‑assembler и далее в исполняемый файл.

## Сборка

Проект написан на C++ (C++17). Для сборки нужны:

- C++‑компилятор (g++/clang++/MSVC),
- NASM (для сборки сгенерированного ассемблера),
- линкер для 32‑битных исполняемых файлов под Windows.

Общий порядок:

```bash
git clone https://github.com/Yula1234/BPMlang.git
cd BPMlang

make
```

Далее компилятор `bpm` принимает `.bpm` файл, генерирует `.asm`, который затем собирается `nasm` и линкуется в exe.

## Статус

Язык и компилятор находятся в активной разработке:

- возможны изменения синтаксиса и ABI,
- оптимизации и модель шаблонов ещё эволюционируют.
